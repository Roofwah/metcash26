How your offers file connects to pictures and the screen

Plain-English guide. The spreadsheet feeds words and numbers into the server. You can either drive card and modal images from the CSV (unified format) or rely on hard-coded maps in the front-end for legacy files.

Two different worlds

The CSV file (backend/offers.csv) is your commercial data: which offer it is, brand, pallet size, SKUs, prices, quantities, and so on.

Legacy format (no image columns): the pictures under frontend/public/products/ are chosen by offer id in frontend/src/config/offersDisplay.ts — “if the offer code is this, show that image.”

Unified format (see backend/offers.unified.template.csv and backend/OFFERS_CSV.md): add columns Logo, Product Image, and Hero with paths like images/myhero.png (files live under frontend/public/). The API exposes logoUrl, productImageUrl, and heroUrl; the app uses those when present, and falls back to the old maps otherwise.

What the server sends to the browser

After the CSV is loaded into the database, the API sends each offer as a bundle of fields. The ones that matter for pictures and placement include:

Field in the appWhere it usually comes from in the legacy CSV
Offer idThe OFFER column (e.g. Energizer 7, Eveready 1)
BrandThe Brand column (used for small brand logos)
Offer groupOffer Group (longer title / range name)
Costs, lines, tiersRest of the columns (descriptions, qty, charge-back, etc.)

If OFFER is blank or the header row is wrong, the app may treat the file as empty and show an error. That is a data problem, not an image problem.

Where the card picture is chosen

There is a single lookup table in code: frontend/src/config/offersDisplay.ts.

It maps a lower-case copy of the offer id to a file path, for example:

  • energizer 7 → big photo on the pre-pack card
  • energizer 1 → pallet card image
  • armorall 4 → car-care style image

If your new offer id is not in that list, the card can still appear, but without that hero image—until someone adds a new line to the map and drops the image file into public/products/.

Where the small brand logo comes from

Still in offersDisplay.ts: the Brand text is scanned for words like “Energizer”, “Eveready”, or “Armor”. If it finds a match, it picks a fixed logo file. So the brand name does not have to be exact, but it needs to contain one of those words for the logo to show.

Why the whole page can look “broken” when you only changed the spreadsheet

Apart from images, another part of the code decides which section of the offers page an offer belongs to: pre-pack strip, lighting, car care, fragrances, or the pallet blocks at the bottom.

That logic is written for specific offer ids and patterns (for example “Energizer 7”, “Energizer 1” through “6”, “Eveready 1” through “3”, and so on). If you rename offers to something completely different, the API might still return them, but they may not appear in any section, so it looks like nothing loaded—even though the server is fine.

In short: the CSV controls what you sell and the numbers. The TypeScript files control which picture and which part of the page each offer id gets. Changing ids without updating those files is the usual reason things feel broken.

Detail screens and pop-ups

The full-page offer view and the modal use more hard-coded checks on the same offer ids and group names (OfferDetail.tsx, OfferDetailModal.tsx). So even if the list shows a card, tapping through might show the wrong image or no extra image until those files are aligned with your new naming.

What to do when you introduce new offers

  1. Keep the header row of offers.csv identical to the working template unless you deliberately change the import code.
  2. Always fill OFFER with a stable code you are happy to use everywhere.
  3. After editing the CSV, restart the API or call the reload endpoint so the database refreshes.
  4. If you use new offer codes: add them to the image map in offersDisplay.ts, add image files under public/products/, and update section rules (and detail/modal code) if the offer should appear in a specific strip or pallet group.
